Foundation

What is a Language Model?

P(next token | context)

"A probability distribution over sequences of tokens"

▸ CS DEFINITION

A statistical model that assigns probabilities to sequences of tokens. Given a context w₁, w₂, … wₙ, a language model estimates P(wₙ₊₁ | w₁…wₙ) — the probability of the next token. Training = maximizing likelihood of observed text.

▸ FINANCE ANALOGY

Like a time-series forecasting model (e.g., AR/ARIMA). Given a history of observations, it predicts the next value. A language model does the same over token sequences instead of price sequences. The "price" is the next word.

★ KEY INSIGHT
Every LLM is, at its core, a next-token predictor. All apparent reasoning, creativity, and knowledge emerges from this single objective applied at massive scale. Understanding this prevents anthropomorphizing the model.

Term 01 / 09

Token

the atomic unit of text

"Not a word — a sub-word piece"

Example: "Quantitative finance" → 3 tokens

Quant itative  finance

~1 token ≈ 4 characters in English

▸ CS DEFINITION

The smallest unit of text an LLM processes. Produced by a tokenizer (e.g., BPE) that splits text into sub-word pieces. A vocabulary of ~50,000 tokens covers most languages. Every input and output is a sequence of token IDs.

▸ FINANCE ANALOGY

Like a tick or bar in market data — the atomic granularity at which information is processed. Just as tick size determines price resolution, token granularity determines how the model "sees" language. Coarser tokenization = lower resolution.

★ KEY INSIGHT
LLM pricing (API cost) is denominated in tokens, not words or characters. Understanding tokenization explains why Korean/Chinese text costs ~2–3× more than equivalent English — denser scripts → more tokens per concept.

Term 02 / 09

Parameters (Weights)

θ ∈ ℝⁿ — the "knowledge" of the model

"Billions of numbers learned from trillions of tokens"

▸ CS DEFINITION

The learnable numerical values of a neural network, updated during training via gradient descent to minimize prediction loss. A "70B model" has 70 billion parameters. Parameters encode all learned patterns — syntax, facts, reasoning heuristics.

▸ FINANCE ANALOGY

Like the coefficients in a fitted factor model. Each parameter is a learned coefficient that weights a feature. Training = running a regression over all human text to fit coefficients. More parameters = a richer, higher-dimensional factor space.

★ KEY INSIGHT
Parameters are static after training — the model doesn't "learn" during inference. What looks like reasoning is pattern-matching over frozen weights. This is why RAG (injecting live data) is necessary for tasks requiring current information.

Term 03 / 09

Transformer

architecture underlying all modern LLMs

"Attention is all you need — Vaswani et al., 2017"

▸ CS DEFINITION

A neural network architecture that processes sequences using self-attention layers. Each token attends to every other token in the context, weighted by relevance. Stacking many attention + feed-forward layers creates a deep model capable of complex reasoning.

▸ FINANCE ANALOGY

Like a full covariance matrix vs. diagonal. Older models (RNNs) processed tokens sequentially — like assuming AR(1). Transformers compute all pairwise token relationships simultaneously — like estimating the full Σ. More expressive, more computationally intensive.

★ KEY INSIGHT
The transformer's key innovation is parallelism: all tokens are processed simultaneously (not one-by-one). This enables training on massive datasets with GPUs. The tradeoff: quadratic memory cost in sequence length — why context windows have limits.

Term 04 / 09

Attention Mechanism

Attention(Q, K, V) = softmax(QKᵀ / √d) · V

"Dynamically weighting which tokens matter for each other"

▸ CS DEFINITION

Each token produces a Query (Q), Key (K), and Value (V) vector. Attention score = dot product of Q with every K, scaled and softmaxed into weights. Output = weighted sum of V vectors. Lets the model dynamically focus on relevant context.

▸ FINANCE ANALOGY

Like a dynamic correlation matrix updated per context. Q is "what am I looking for?", K is "what do I offer?", V is "what information do I carry?". Softmax weights ≈ normalized correlations. The model recomputes these weights for every new input.

★ KEY INSIGHT
Attention solves the long-range dependency problem. Earlier models forgot context after ~100 tokens. Attention lets "stock" in position 1 directly inform "price" in position 500 with full weight — no decay. This is why LLMs handle long documents well.

Term 05 / 09

Pre-training

self-supervised learning on massive corpora

"Learning world knowledge by predicting masked or next tokens"

▸ CS DEFINITION

Training a model on a massive unlabeled corpus (trillions of tokens from web, books, code) using a self-supervised objective: predict the next token (GPT-style) or masked tokens (BERT-style). The model learns language, facts, and reasoning patterns with no human labels required.

▸ FINANCE ANALOGY

Like unsupervised factor discovery (PCA on returns) — the model finds latent structure without labeled outcomes. Or: training on all Bloomberg news ever published to learn "the grammar of finance" before fine-tuning on specific prediction tasks.

★ KEY INSIGHT
Pre-training is astronomically expensive (GPT-4 estimated ~$100M+). Fine-tuning is cheap by comparison. This cost structure creates a natural separation: frontier labs bear pre-training costs; practitioners add value through fine-tuning and prompting.

Term 06 / 09

Fine-tuning & RLHF

supervised adaptation + reinforcement learning from human feedback

"Teaching the model to be helpful, harmless, and honest"

▸ CS DEFINITION

Fine-tuning: continuing training on a smaller, task-specific dataset to adapt a pre-trained model. RLHF: human raters rank model outputs; a reward model is trained on rankings; the LLM is updated via RL to maximize the reward — aligning outputs with human preferences.

▸ FINANCE ANALOGY

Like transfer learning in quant finance: start with a pre-trained macro model, fine-tune on your specific asset class. RLHF ≈ incorporating human expert judgment as a reward signal — similar to using analyst ratings to adjust model output rankings.

★ KEY INSIGHT
RLHF transforms a raw next-token predictor into an assistant that follows instructions. Without it, the model would complete prompts literally ("List 3 stocks" → "1. Apple 2. Microsoft 3…") rather than helpfully. It is what makes ChatGPT feel like a chatbot, not a search engine.

Term 07 / 09

Temperature & Sampling

P'(token) ∝ P(token)^(1/T)

"Controlling randomness in generation"

▸ CS DEFINITION

At each step, the model outputs a probability distribution over all tokens. Temperature T scales the distribution: T → 0 = greedy (always pick the highest probability token); T → 1 = sample proportionally; T > 1 = more uniform / random outputs.

▸ FINANCE ANALOGY

Like a risk aversion parameter. Low temperature = highly risk-averse decision maker who always picks the "safest" (most probable) option. High temperature = risk-seeking, explores low-probability outcomes. For financial analysis, use T ≈ 0 for deterministic, reproducible outputs.

★ KEY INSIGHT
Temperature is the single most important inference parameter for practitioners. T=0 for factual retrieval and analysis (reproducible). T=0.7–1.0 for brainstorming and drafting. Never use high temperature for compliance-sensitive financial outputs — variance is a liability.

Term 08 / 09

Prompt Engineering

system prompt + few-shot examples + user query

"The interface between human intent and model behaviour"

▸ CS DEFINITION

Crafting the input text (prompt) to steer model outputs without changing weights. Techniques: zero-shot (instruction only), few-shot (instruction + examples), chain-of-thought (ask the model to reason step-by-step before answering). The system prompt sets persistent context.

▸ FINANCE ANALOGY

Like query construction in a Bloomberg function. The same terminal has different outputs depending on how you specify the function and parameters. Prompt engineering = knowing which {FIELD} and syntax yields useful data. Garbage in, garbage out applies equally.

★ KEY INSIGHT
Chain-of-thought prompting ("think step by step") reliably improves accuracy on multi-step reasoning tasks — the model's intermediate tokens act as working memory. For financial calculations or logic chains, always elicit reasoning before the final answer.

Term 09 / 09

Hallucination

confident generation without grounding

"The model completes the pattern, whether true or not"

▸ CS DEFINITION

An LLM generates text that is fluent and confident but factually wrong or fabricated. Root cause: next-token prediction optimizes for plausibility, not truth. The model has no internal truth-verification mechanism — it cannot distinguish remembered fact from plausible interpolation.

▸ FINANCE ANALOGY

Like an analyst who extrapolates a DCF model beyond the data — the output is internally consistent and well-formatted, but the inputs were fabricated. Or: a model that generates a plausible-looking ISIN or filing date that doesn't exist. Looks right; is wrong.

★ KEY INSIGHT
Hallucination rate correlates with query specificity and data recency. LLMs are most reliable on common, well-documented facts; least reliable on specific numbers, dates, names, and recent events. Always verify quantitative claims from LLM outputs against primary sources — this is where RAG adds critical value.
